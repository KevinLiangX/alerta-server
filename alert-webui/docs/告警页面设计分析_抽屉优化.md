# 告警详情页侧滑抽屉（Right Drawer）可行性分析

## 1. 原有组件 (AlertDetail.vue) 结构分析
目前 `AlertDetail.vue` 是按照全屏页面设计的，但经过深入分析源码，它**天然非常适合**直接无缝嵌入侧边抽屉中：
- **百分比网格系统**：左侧 Label 和右侧数据使用了 `xs3` (25%) 和 `xs6` (50%) 等原生的 Vuetify Flexbox。在 600px - 800px 宽度的抽屉中，它会自动等比例缩紧，布局不会错乱。
- **历史记录表格 (History)**：组件中已经写好了响应式断点控制（如 `hidden-sm-and-down`），放入稍微窄一点的抽屉里，它自动会隐藏冗余列，保留核心流。
- **RawData JSON 视图**：这部分刚刚已经被我们优化为 `<pre>` 换行展示，因此也不会把抽屉撑爆。
- **顶部操作按钮栏**：Action 按钮（Refresh, Watch, Ack...）全部是 `icon` 按钮，占用体积极小，就算由于宽度不够变成两行，视觉也能接受。

**结论：无需大规模重写 CSS，只需更改外层容器即可实现抽屉化。**

## 2. 实施方案大纲

### 第一阶段：App.vue 顶部导航栏瘦身
- 彻底移除 `App.vue` 中的 `<v-toolbar>`。
- 将原本属于列表操作的三个按钮（刷新、过滤弹窗、CSV下载）迁移到 `Alerts.vue` 中 `severity-strip` (状态分布条) 的最右侧，使用 `v-spacer` 将它们推到右边。

### 第二阶段：告警列表交互重构（侧滑抽屉）
- 在主界面内引入 `<v-navigation-drawer right temporary>` 或一个定制的滑动面板。
- 拦截 `AlertList.vue` 中对于行元素的点击事件：原本是触发 `$router.push('/alert/:id')`，现修改为打开右侧 Drawer，并将 `id` 传给里层的 `<alert-detail>` 组件。
