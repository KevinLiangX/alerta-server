# Alerta WebUI 告警页面设计分析

Alerta WebUI 的主告警页面位于 `src/views/Alerts.vue`，并通过引入子组件 `src/components/AlertList.vue` 来呈现核心的告警列表数据。它是一个典型的支持自动刷新、按环境过滤和多维度搜索的监控大盘界面。

## 页面整体布局 (Layout)

整个告警页面 (`Alerts.vue`) 的界面的顶层布局非常清晰，主要分为以下几个部分：

### 1. 顶部操作栏与环境切换区 (Tabs & Toolbar)
*   **环境 Tab 页 (v-tabs)**: 页面的最上方使用了 Vuetify 的 `v-tabs` 组件。它根据后端返回的 `environments` (比如 Production, Development) 动态生成 Tab 标签。
    *   **特殊 Tab**: 始终固定存在一个名为 "ALL" 的 Tab，用于展示无过滤的所有环境告警。
    *   **数量显示**: 每个 Tab 名字后带有括号数字，例如 `Production (5)`，直观地表明该环境下目前有多少条未关闭的开放告警。
*   **过滤按钮 (Sidesheet Toggle)**: Tab 右侧有一个筛选图标 (`filter_list`)，点击会呼出右侧滑动面板 (`AlertListFilter.vue`)，用于高级条件搜索。
*   **菜单按钮 (More Vert)**: 最右侧有三个点的更多功能菜单，展开后提供：
    *   **展示 / 隐藏顶层面板 (Panel)**: 也就是 Indicator 区块。
    *   **显示密度设置 (Display Density)**: 控制表格的紧凑度 (Comfortable vs Compact)。
    *   **导出为 CSV (Download as CSV)**: 利用 `export-to-csv` 库将当前的告警数据导出。

### 2. 摘要指标面板 (Indicator Panel / Expandable Panel)
*   可以通过更多菜单展开的区域，包含各种**统计指标图块 (Indicators)**。
*   这个区域会遍历 `$config.indicators.queries`（一般通过 `config.json` 注入），展示类似“总严重告警数”、“总网络告警数”这种数字 Dashboard，使用组件 `AlertIndicator.vue` 渲染。

### 3. 右侧侧边栏 (Sidesheet / Filter Dashboard)
*   呼出后的快速筛选面板 (`AlertListFilter.vue`)
*   包含文本搜索、按服务(Service)、客户(Customer)、分组(Group)、甚至状态(Status) 和时间段下拉的多维度筛选表单。
*   选中的筛选条件会被同步到 Vuex (`$store.state.alerts.filter`) 并在 URL 路由上打上 Hash 标记，实现**状态保持和可复制分享链接**。

## 核心数据渲染：告警列表 (`AlertList.vue`)

下方区域使用 `<keep-alive>` 包裹了重头戏组件 `AlertList.vue`。这个组件并不是简单的列表，而是一个功能极其丰富的**数据表格 (v-data-table)**。

### 1. 视觉与排版 (Styling & Formatting)
*   **行颜色 (Row Background):** 这个是 Alerta 的一个重要设计。表格**每一行 (TR)** 的背景色完全取决于该条告警的严重程度 (`severity`)。
    *   依赖 `severityColor(severity)` 函数，它读取了 `colors` 配置（如 critical 为暗红色，warning 为蓝色或黄色等）。
    *   这种设计在视觉上极具冲击力。如果有致命级别告警，大盘上一眼望去便是刺眼的红色。
*   **列自定义排布:** 表头的列并不是写死的，而是读取 `$config.columns` 来动态生成 `<td>`，这就允许管理员在外部 `config.json` 强定义大盘要显示哪些字段（例如 `['severity', 'status', 'lastReceiveTime', 'environment', 'service', 'resource', 'event', 'value']`）。

### 2. 重要字段解析
表格中渲染的字段包含了很多状态和交互逻辑：
*   **趋势箭头 (Trend Arrow):** 行最左侧，如果在前端计算得出它的严重级别正在升高 (`moreSevere`)，会显示向上箭头 ⬆️；反之则显示向下或横杠。
*   **严重性标签 (Severity):** 使用 `label-critical`, `label-major` 等自定义 class，使用大写加粗样式的徽章展现。
*   **时效性计算:** `timeout` (心跳超时阈值) 和 `timeoutLeft` (距离超时还剩多久) / `duration` 字段的动态渲染，背后依靠 `moment.js` 计算。

### 3. 悬浮操作菜单 (Action Buttons / Hover Menu)
这是该页面非常重的一个交互设计：
*   默认情况下行的右侧显得干净。
*   但是通过 `.action-buttons` 以及对告警状态的判断 `isOpen`, `isAcked`, `isShelved` 判断，向用户暴露了快捷操作按钮。
*   只对对应状态显示正确的按钮：
    *   **如果它是 打开状态 (Open)**：显示打钩图标 `ackAlert` （让运维认领、确收告警）；显示时钟图标 `shelveAlert` (屏蔽/暂存告警)。
    *   **如果它是 已确收 (Acked)**：显示撤销按钮 `takeAction('unack')`。
    *   **如果它是 被暂存 (Shelved)**：显示恢复按钮 `takeAction('unshelve')`。
    *   **关注操作:** 提供对单一告警点“小眼睛”图标的 watch / unwatch 操作。

## 数据流向与刷新机制 (Data Flow & State)

1.  **全局状态 Vuex 管理**: `<Alerts>` 组件本身几乎不维护告警数据，只维护 UI 状态（如弹窗开启/关闭、Tab标签切换）。所有的告警获取 `getEnvironments()`, `getAlerts()` 都是通过 `this.$store.dispatch('alerts/...', ...)` 触发，并由 Vuex 的 getters 实时响应渲染到表单。
2.  **自动轮询 (Polling)**: 依靠 `refreshAlerts()` 的定时器 `setTimeout(..., this.refreshInterval)`，周期性（默认通过配置，一般是 5-10 秒）向后端拉取最新的告警列表。
3.  **声音提醒 (Audio Reminder)**: 如果配置了 `audioURL` 且有新的且状态为 Open 的告警 (`isNewOpenAlerts`)，前端会悄悄播放一段警示提示音 `this.$refs.audio.play()`。
4.  **防抖处理 (Debounce)**: 在 `AlertList.vue` 中点击操作按钮（比如 Ack 告警, Shelve 告警），使用了 `lodash/debounce` 打上了防抖处理 (`200ms` 判定），以防止用户手抖快速点击同一个告警产生多次重复的发包。

## 总结
Alerta 的大盘设计遵循着**强视觉提示**、**高信息密度**（可调）、**快速筛选与隔离（基于环境）**的产品思路，技术上通过 Vuex 的强状态同步来实现从外部 JSON 渲染灵活配置的 Dashboard。
